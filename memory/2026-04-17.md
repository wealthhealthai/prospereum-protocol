### BlockApex Final Report ‚Äî 3 Additional Findings (all FIXED)

**`4ca5bfc` ‚Äî Additional fixes #1 #2 #3:**

**Additional #1 (HIGH) ‚Äî StakingVault settlement cursor (retroactive theft / reward loss / insolvency)**
- Root cause: O(N) epoch-cursor loop (`_settleFinishedEpochs`) had two defects:
  1. `lastSettledEpoch` starts at 0 for new users ‚Üí retroactive theft (credited historical epochs)
  2. `MAX_SETTLE_EPOCHS=52` could advance cursor past epochs with stale balances ‚Üí reward loss
- Fix: True Synthetix cumulative accumulator (`_updatePending`, O(1))
  - `cumulativePSRERewardPerToken` + `cumulativeLPRewardPerToken` incremented per epoch at `distributeStakerRewards()`
  - Per-user `userPSRERewardPerTokenPaid` / `userLPRewardPerTokenPaid` auto-initialize new users to current cumulative ‚Üí only future epochs credited
  - No gas cap needed ‚Äî O(1) is safe for any epoch gap
- Regression tests added: `test_POC1_newStakerCannotStealHistoricalRewards`, `test_POC2_noRewardLossOnUnstakeAfterManyEpochs`

**Additional #2 (HIGH) ‚Äî autoFinalizeEpochs √ó scarcityCeiling DoS**
- Root cause: `scarcityCeiling = E_scarcity √ó K` could exceed `E0_MAX` (= `MAX_MINT_PER_EPOCH = 25200e18`) ‚Üí `mintForEpoch` reverts ‚Üí DoS on all vault operations
- Fix: Clamp `scarcityCeiling = min(E_scarcity √ó K, E0_MAX)` before computing budget

**Additional #3 (MEDIUM) ‚Äî Staker share minted when zero stakers**
- Root cause: `P_stakers` minted even when no PSRE or LP staked ‚Üí tokens orphaned on contract, `S_EMISSION` budget permanently consumed with no recovery
- Fix: Check `epochTotalPSREStaked + epochTotalLPStaked` after snapshot; if both 0 ‚Üí `P_stakers = 0`, no mint
- `IStakingVault` extended: `epochTotalPSREStaked(epochId)` + `epochTotalLPStaked(epochId)` view functions
